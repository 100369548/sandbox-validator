## 核心服务栈

### PassWall2（透明代理）
- **模式**：xray + chinadns-ng + nftables TPROXY
- **智能分流**：HK优先(主) → 故障按延迟切备份(9节点leastLoad) → 600秒回切
- **DNS 分流**：chinadns-ng (国内3个DNS, 国外1.1.1.1 DoH)
- **关键端口**：SOCKS :1081, TPROXY :1041, xray DNS :15353, chinadns-ng :15354
- **nftables 表**：`inet passwall2`
- **策略路由**：table 999 (fwmark 0x50535732)
- **订阅更新**：已禁用自动，改为每12小时 cron 手动触发
- **autoswitch**：15秒探测 / 2秒超时 / gstatic.com 探测

### SQM QoS
- CAKE on eth0, 800Mbit, diffserv4 + triple-isolate + nat

### DHCP
- 范围：192.168.100.100-249, 租约 12h
- ~20 台在线设备

## 已知问题 & 修复机制

### nftables 规则周期性丢失
- **现象**：PassWall2 规则/订阅每小时更新，可能触发规则丢失
- **影响**：仅科学上网失效，国内正常
- **自动修复**：`/root/fix_passwall2.sh` + cron `*/5 * * * *`
- **修复内容**：nft 表、路由表、xray/chinadns 进程、DNS 配置

### DNS 优化
- 当前配置含 3 个国内 DNS 上游 + 3 层缓存（dnsmasq→dnsmasq_default→chinadns-ng）
- chinadns-ng 缓存 4096 条，过期后 stale 3600 秒可用

## 校验工具

### Go 命令行版（v4.8 最新 — 主力工具）
- **源码**: `sandbox_validator_go/main.go`（v4.8）
- **Mac 可执行文件**: `~/Desktop/沙盘校验工具_mac`（8.2MB）

### 校验规则详情（v4.8 最新）
| 规则 | 说明 |
|------|------|
| 枚举 | 省份/进度/状态/**门店类型(15种)**/运营商/国补/城乡/华为属性/OPPO层级/vivo层级/小米层级 |
| 数字 | capa/area/销量/人数 0-999999 |
| 手机号 | 11位 1[3-9]开头；**支持填"无"** |
| 姓名 | 支持填"无"（联系人/老板名/操盘手名） |
| 门店编码 | 字母开头(CNS) 或 **46**开头 或 纯数字≥8位 |
| 引用 | 地理编码/商圈编码 必须存在于对应信息Sheet |
| 渠道编码 | 纯数字或"无" |
| 勾稽1 | **K = T+U+V = W+X+Y+Z+AA+AB** (capa = 价位段 = 品牌) |
| 勾稽2 | **M=闭店 + 全品牌销量为0 → 直接通过** |
| 勾稽3 | **TOP1/TOP2/TOP3 跨品牌可重复**（AK=36/AL=39/AM=42/AN=45/AO=48/AP=51） |
| 勾稽4 | 无陣地(U+9635) 非 U+9663 — 修正编码错误 |
