## 路由器智能分流配置

### 背景
- 用户希望国外流量优先走香港，香港故障时按延迟自动切换到其他节点

### 配置方案
- **国内**：`geosite:cn` → 直连
- **国外**：默认 → `balancer-hk`（香港2节点）
- **故障切换**：15秒探测 / 2秒超时 → `balancer-backup`（9节点 leastLoad）
- **回切**：600秒后自动切回香港

### 节点列表
```
香港组(主)：hk-telecom, hk-bgp
备份组(9节点)：jp-telecom, jp-bgp, jp-home, sg-telecom, sg-bgp, tw-telecom, tw-bgp, us-telecom, us-bgp, us-home
```

### 实施
1. 生成新 xray 配置 `SOCKS_XpV6OVrm.json`
2. SCP 到路由器并重启 PassWall2
3. 实测验证：google 1.51s / baidu 0.13s ✅

### 配置持久化
- **问题**：PassWall2 订阅更新每小时运行，会覆盖新配置
- **解决**：禁用自动订阅更新 + 添加每12小时手动触发 cron
- **最终 cron**：`0 */12 * * * lua /usr/share/passwall2/subscribe.lua`
